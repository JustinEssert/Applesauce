Now that you know a little about what Git & GitHub are, let's see how they look in action!

Part 1: Configuring Git

The first thing that we'll do is setup git on your computer, this only has to be done once when you first install git.

  1. We need to start off by telling git who you are. To do so execute the following commands in the terminal, replacing "bucky badger" with your first and last name, and "bbadger@wisc.edu" with the email that you used to create your GitHub account.
    • git config --global user.name bucky badger
    • git config --global user.email bbadger@wisc.edu
  2. This step is optional:
    Next, you may want to save your GitHub email and password to your computer, to do so execute "git config --global credential.helper store". Note:This tells git that you would like to save your GitHub credentials the next time you use them, but it will not prompt you for the credentials. You will still have to enter in your GitHub credentials once.
    • git config --global credential.helper store
  3. And that's it! You've successfully configured git on your computer.

Part 2: Creating a Git Repository

Next, we want to create a git repository (or repo). A git repo is just a folder that we will use for a git project.

  1. In the terminal, create a new folder called git-tutorial/ and navigate into it.
    • mkdir git-tutorial
    • cd git-tutorial
  2. Within the git-tutorial/ folder, use git init to create a git repository.
    • git init
  3. You should receive a message that says you have initialized an empty repository in the git-tutorial folder.

Part 3: Creating a GitHub Repository & Linking it to our Git Repository

After creating a git repo, we will want to create a GitHub repo. A GitHub repo is just a online version of our git repo that we can use to backup and share our code.

  1. Log in to GitHub
  2. Click on the button to Create a New Repository.
  3. On the next page, give your repository a name, mark it as private, then hit Create Repository.

    Note: Creating a public repository allows anyone be able to find your code. This is ok many cases, but for now we will just make a private repo (you can delete it after this tutorial if you want).
  4. GitHub will now bring you to your repository's webpage, copy down the URL for the page, you will need this in the next step.

  5. Now we need to tell our git repo that the GitHub repo exists, to do that we will go back to the terminal and use the git remote add command as seen below. Replace the url in the example with the url that you copied down in the last step.
    • git remote add origin https://github.com/buckybadger/git-tutorial
    Note:This command gives your github url the shortened name of origin (you can name it whatever you want by replacing "origin" in the above code) for ease of use.

Part 4: Making your First Commit.

Now that your repo is all set up we will want to make an initial commit. A commit is the process of saving your code to a git repository.
Note: Just like you should save a file often while working on it, you should commit your code often as well.

  1. Within the git-tutorial/ folder, create a file called example.html.
  2. Open up example.html and copy in the following code:
    example.html
    1. <h2>Hello World!</h2>
    2. <p>
    3. I'm doing a git tutorial!
    4. </p>
  3. Save the file and navigate back to the git-tutorial/ folder in the terminal.
  4. Next, we will add the example.html file to our git repository. This tells git that it should track this file.
    • git add example.html
  5. Now we can use the git commit command to create a commit. Note: The -m flag allows you to create a message describing what you did since the last commit in quotations, you should do this on every commit!
    • git commit -m "created a example.html file"
  6. We can verify that the commit was successful by using the git log command, which shows us a log of all the previous commits.
    • git log
    You should see a few lines which say the commit id (a long string of characters), the author of the commit (you), the date of the commit (today), and the message that you added to the commit.
    • commit eeba2040c5af96e20bfaa3a4448e8810fe6a249b (HEAD -> master)
    • Author: bbadger@wisc.edu
    • Date: Mon Jan 1 11:00:00 2018 -0500

    • created a example.html file
    Note: If the output text is too large to fit in your terminal window you will need to type "Q" to exit.
  7. After we commit our code to our git repo, we should upload it to our GitHub repo. This can be done through a command called git push, as seen below.
    • git push origin master
    Note: Here we are specifying "origin" as the source, if you decided to name your github url something different, you will need to change "origin" to the name you created.
  8. Refresh your GitHub repository page and you should see your uploaded code!
