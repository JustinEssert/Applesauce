Now that you know a little about what Git & GitHub are, let's see how they look in action!

  1. First off, open a terminal window (there are ways to use Git outside of using the terminal, but it is much easier and faster, once you get used to it, to use the terminal).
  2. Create a new folder called git-tutorial/ and navigate into it.
    • mkdir git-tutorial
    • cd git-tutorial
  3. Within the git-tutorial/ folder, create a file called example.html.
  4. Open up example.html and copy in the following code:
    example.html
    1. <h2>Hello World!</h2>
    2. <p>
    3. I'm doing a git tutorial!
    4. </p>
  5. Save the file and navigate back to the git-tutorial/ folder in the terminal.
  6. Within the git-tutorial/ folder, use git init to create a git repository.
    • git init
    You should receive a message that says you have initialized an empty repository in the git-tutorial folder.
  7. Execute the following commands in the terminal, replacing "bucky badger" with your first and last name, and "bbadger@wisc.edu" with the email that you used to create your GitHub account.
    Note:The last command saves the username and password for your github account onto your machine, if you do not want to do this, skip the last command.
    • git config --global user.name bucky badger
    • git config --global user.email bbadger@wisc.edu
    • git config --global credential.helper store
  8. Log in to GitHub and click on the button to Create a New Repository. On the next page, give your repository a name, mark it as private, then hit Create Repository.

    Note: Creating a public repository allows anyone be able to find your code. This is ok many cases, but for now we will just make a private one (you can delete it after this tutorial if you want).
  9. GitHub will now bring you to your repository's webpage, copy down the URL for the page, you will need this in the next step.

  10. One more configuration step that we need to do is tell git that we have created a online repository for our code. This is something that you only have to do once per repository. To do so, we will use git remote add, replacing "https://github.com/buckybadger/git-tutorial" with the URL that you copied down in the previous step.
    • git remote add origin https://github.com/buckybadger/git-tutorial
    Note:This command gives your github url the shortened name of origin (you can name it whatever you want by replacing "origin" in the above code) for ease of use.
  11. Let's take a break here, everything above this step was configuring git, GitHub, and your repository. These are not things that you will have to do very often, so just remember that they exist, but feel free to forget how to do it (you can always look up how to do these steps again later).
  12. Next, we will add the example.html file to our git repository.
    • git add example.html
  13. Once we add the example.html file, we can create a commit. This is the process of saving your code to a git repository.
    Note: The -m flag allows you to create a message describing what you did since the last commit in quotations, you should do this on every commit!
    • git commit -m "created a example.html file"
  14. Now we can double check that this worked by running the git log command.
    • git log
    You should see a few lines which say the commit id (a long string of characters), the author of the commit (you), the date of the commit (today), and the message that you added to the commit.
    • commit eeba2040c5af96e20bfaa3a4448e8810fe6a249b (HEAD -> master)
    • Author: bbadger@wisc.edu
    • Date: Mon Jan 1 11:00:00 2018 -0500

    • created a example.html file
    Note: You will need to type "Q" to exit the git log window
  15. Finally, we need to upload our code to GitHub. This can be done through a command called git push, as seen below.
    • git push origin master
    Note: Here we are specifying "origin" as the source, if you decided to name your github url something different, you will need to change "origin" to the name you created.
  16. Refresh your GitHub repository page and you should see your uploaded code!
