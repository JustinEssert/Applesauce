Great! You've worked through configuring git and started a new git project. Now we'll explore more into commits and start learning about branches.

  1. Before we do anything else, we should run git status to double check that everything that we've done has been recorded by git.
    • git status
    You should see the message below, this means that git is up to date. We'll figure out what some of the other terminology here means in a short while.
    • On branch master nothing to commit, working tree clean
  2. Let's open up our example.html file and modify the third line to say "Git is a great tool!".
    example.html
    1. <h2>Hello World!</h2>
    2. <p>
    3. Git is a great tool!
    4. </p>
  3. Now again, we need to tell git that we've modified this file, so first we run a "git add" command.
    Last time we ran "git add example.html", but in general, you will not want to add each file individually, it's just too slow. Instead let's get in the habit of using "git add .", which adds all the files in our current directory (as well as any sub-directories).
    • git add .
  4. Now if we run "git status" again, we'll see that git recognizes that example.html has been modified and is waiting for a commit.
    • git status
  5. Now let's commit our change to git.
    • git commit -m "updated hello world message"
  6. Unless you have a reason for not doing so, it is often best to just get in the habit of pushing your code at the same time as you commit (remember that a push is like uploading to a shared Google Drive folder).
    • git push origin master
  7. Now if we run "git log" again, we should see two commits.
    • git log
    You're output should look something like this:
    • commit 91aa1d3e655a5bbc5a53aff8cc9c80bb5f48802e (HEAD -> master, origin/master)
    • Author: bbadger@wisc.edu
    • Date: Mon Jan 1 12:00:00 2018 -0500

    • updated hello world message
    • commit eeba2040c5af96e20bfaa3a4448e8810fe6a249b
    • Author: bbadger@wisc.edu
    • Date: Mon Jan 1 11:00:00 2018 -0500

    • created a example.html file
  8. Now let's say that, for whatever reason, we need to roll-back our code to the way we had it in our first commit. To do this, let's grab the commit id that from our first commit (this is displayed in the output of git log).
    Note: In the case above, the commit id is 'eeba2040c5af96e20bfaa3a4448e8810fe6a249b' but your commit's id will almost certainly be different, so make sure you get the commit id directly from the git log output.
  9. Once you have your first commit's id, let's go to that code by using the command 'git checkout' followed by the commit id.
    • git checkout eeba2040c5af96e20bfaa3a4448e8810fe6a249b
  10. Now if we look at the text in example.html, line 3 has the old message so we rolled back correctly.
    example.html
    1. <h2>Hello World!</h2>
    2. <p>
    3. I'm doing a git tutorial!
    4. </p>
  11. Congrats, you just learned how to roll-back code to a previous version! Take a minute to review what we've done so far before moving to the next step where we introduce branches.
  12. Now we are going to create a new 'branch' for this rolled back version, we'll get more into what this means in a few steps.
    Note: The -b tag on git checkout tells git that you want to create a new branch, in this case called 'new-branch' (clever, I know).
    • git checkout -b new-branch
  13. Now let's update our hello world message again, this time let's change the title to "Hello Galaxy!".
    example.html
    1. <h2>Hello Galaxy!</h2>
    2. <p>
    3. I'm doing a git tutorial!
    4. </p>
  14. Like always, we now need to add our updated files and commit our changes.
    • git add .
    • git commit -m "changed title in example.html"
  15. Now, if we want to push our code, we must also tell git that it needs to push to the new branch rather than the original one.
    Before, we executed the command "git push origin master" where "origin" refers to the github url that we saved and "master" refers to the branch (so our original branch is called "master"). To push our code to our new branch, we must exectute the command "git push origin new-branch".
    • git push origin new-branch
