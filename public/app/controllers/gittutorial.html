Git & Github Tutorial

Now that you know a little about what Git & GitHub are, let's see how they look in action!

Part 1: Setting up a Git Repository
  1. First off, open a terminal window (there are ways to use Git outside of using the terminal, but it is much easier and faster, once you get used to it, to use the terminal).
  2. Create a new folder called git-tutorial/.
  3. Within the git-tutorial/ folder, create a file called example.html.
  4. Open up example.html and copy in the following code.

    <h2>Hello World!</h2>
    <p>
    I'm doing a git tutorial!
    </p>

  5. Save the file navigate to git-tutorial/ in the terminal.
  6. Within the git-tutorial/ folder, do the following command to create a git repository.
    computer:git-tutorial user$ git init
    You should receive a message that says you have initialized an empty repository in the git-tutorial folder.
  7. Log in to GitHub and click on the button to Create a New Repository . On the next page, give your repository a name and mark it as private, then hit Create Repository.
    Note:Creating a public repository allows anyone be able to find your code. This is ok many cases, but for now we will just make a private one (you can delete it after this tutorial if you want).
  8. GitHub will now bring you to your repository's webpage, copy down the URL for the page, you will need this in a few steps.
  9. Navigate to back to the terminal window (you should still be inside the git-tutorial/ folder). Insert the following commands into the terminal, replacing "first last" with your first and last name, and "email" with the email that you used to create your GitHub account.
    Note:This will only have to be done once on your computer, this step involves telling git who you are and saving your user name and password to the computer. If you do not want to save your username/password, only do the first two commands.
    computer:git-tutorial user$ git config --global user.name first last
    computer:git-tutorial user$ git config --global user.email email
    computer:git-tutorial user$ git config --global credential.helper store
  10. One more configuration step that we need to do is tell git that we have created a online repository for our code. This is something that you only have to do once per repository. Replace https://github-repository-url with the URL that you copied down in a previous step.
    computer:git-tutorial user$ git remote add origin https://github-repository-url.
  11. Let's take a break here, everything above this step was configuring git, GitHub, and your repository. These are not things that you will have to do very often, so just remember that they exist, but feel free to forget how to do it (you can always, and will have to sometimes, look up how to do these steps when you need to).
  12. Next, we are going to do something called a git commit, this is the process of saving your code to a git repository. You can ommit your code with the following code:
    Note: The -m flag allows you to create a message describing what you did since the last commit in quotations, you should do this on every commit!
    computer:git-tutorial user$ git commit -m "created a example.html file"
  13. Finally, we need to upload our code to GitHub. This can be done through a command called git push, as seen below.
    computer:git-tutorial user$ git push origin master
  14. Refresh your GitHub repository page and you should see your uploaded code!
